# Задача

Разработать класс `TaskManager` для управления задачами в системе. Класс должен хранить задачи, обеспечивать доступ к ним и поддерживать операции управления.

---

## Структура задачи

Каждая задача должна содержать следующие поля:

- `id` — уникальный идентификатор (`uuid.UUID`)  
- `title` — название задачи (str)  
- `description` — описание задачи (str)  
- `priority` — приоритет задачи (int, например, 1–5)  
- `tags` — множество тегов (`frozenset[str]`)  
- `created_at` — время создания задачи (float, UNIX-время)  
- `updated_at` — время последнего обновления задачи (float, UNIX-время)  
- `status` — статус задачи (Enum), возможные значения:  
  - `PENDING` — ожидает  
  - `IN_PROGRESS` — в процессе  
  - `DONE` — выполнено  
  - `CANCELLED` — отмена  
  - `ERROR` — ошибка  
- `payload` — полезная нагрузка задачи (словарь `dict[str, Any]`) с возможностью шифрования данных

---

## Бизнес-требования к классу `TaskManager`

1. Класс должен позволять **добавлять задачи**.  
2. Класс должен обеспечивать **поиск задач** по тегам, приоритету и статусу выполнения.  
3. Класс должен позволять **фильтровать задачи** по различным критериям (например, время создания, время обновления, статус, приоритет).  
4. Класс должен позволять **удалять задачи** по времени создания, статусу или другим критериям.  
5. Класс должен предоставлять **подсчёт общего количества задач**.  
6. Класс должен предоставлять **статистику по тегам** (например, количество задач на каждый тег).  
7. Класс должен предоставлять **статистику по приоритетам** (например, количество задач каждого приоритета).  
8. Каждая задача должна иметь **уникальный идентификатор `id` (uuid)**.  
9. Класс должен обеспечивать **эффективное хранение и управление коллекцией задач**.  
10. Класс должен обеспечивать **сохранность данных** при чтении и записи задач во внешнее хранилище.
